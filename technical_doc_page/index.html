Importing Python packages

What you need to do to get from knowing about a package to importing said package in Python.

Python Imports

The import command is a basic function of the Python language framework that allows you as a user to access Python built-in packages as well as unique packages that are not part of Python, usually installed using pip.

We will be looking into everything around using import for a specific Python package called Pandas, which is used for data transformations and manipulations. We'll start with installing the package.

Python Package Installs

There are a lot of ways to install Python packages. You could use Anaconda, build from wheel, or use the most common method, the pip install method. We will be using the last method

The most common syntax for using pip is as follows

pip install pandas

What if you want to install a specific version, let's say version 2.2.1? Then you specify it as follows

pip install pandas==2.2.1

But what if you have python2.7 and 3.10? Then you specify it as follows

python3 -m pip install pandas==2.2.1
The Import

We've installed Pandas now, so create a .py or .ipynb file in your IDE (like vsCode or PyCharm) and write the following line

import pandas

You can even use a shorthand to refer to when using it in your code, like so

import pandas as pd

From Import

Another way to import packages is to import specific parts of it, using from. Lets say you don't need the full Pandas package, and only some of the definitions in it, you'd use the from ... import ... syntax. Lets say you only need the DataFrame class of pandas, your import would look like this

from pandas import DataFrame

or

from pandas import DataFrame as df

Now you can install a package and import it or a portion of it!

Troubleshooting

Let's say you struggle to get a package installed using pip, you can check where the error lies. Potential problems can be that

  • the version you specified is not compatible with the python interpreter version you are using. It can cause dependency issues if you have older packages installed that relied on the older version of pandas.
  • the interpreter's pip path is not picked up
These are two basic things you can check, but there's a plethora of reasons why the install failed. A surefire way to get some answers or help is to copy the error and error code (if present), paste it in Google or your search engine of choice (even chatGPT if you have an account) and seeing what pops up. Stackoverflow links are usually a good place to start

For import fails, the same applies as above, and you just have to search for the options to test. You can try the following:

  • Check if the package installed to the wrong Python version
  • Is the Python version you want to use in your path
  • Is the right Python version selected as interpreter/kernel for your code file.
These are the first three things I'd check.
